FEEDBACK MODEL
==============

Schema Definition:
```prisma
model Feedback {
  id          String   @id @default(cuid())
  siteId      String
  site        Site     @relation(fields: [siteId], references: [id], onDelete: Cascade)
  rating      Int
  comment     String?
  email       String?
  likesCount  Int      @default(0)
  isPublished Boolean  @default(false)
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  @@index([siteId])
  @@index([likesCount])
  @@index([createdAt])
}
```

Database Table Structure:
- id: TEXT PRIMARY KEY (CUID)
- siteId: TEXT NOT NULL (Foreign Key → Site.id, CASCADE)
- rating: INTEGER NOT NULL
- comment: TEXT (nullable)
- email: TEXT (nullable)
- likesCount: INTEGER DEFAULT 0
- isPublished: BOOLEAN DEFAULT false
- createdAt: DATETIME DEFAULT CURRENT_TIMESTAMP
- updatedAt: DATETIME (auto-updated)

Indexes (3 total):
1. Feedback_siteId_idx - For filtering by site
2. Feedback_likesCount_idx - For sorting by popularity
3. Feedback_createdAt_idx - For chronological queries

Relationships:
- Many-to-One: Feedback → Site (onDelete: Cascade)
- When a Site is deleted, all its Feedback records are deleted

TAG MODEL
=========

Schema Definition:
```prisma
model Tag {
  id        String   @id @default(cuid())
  name      String   @unique
  slug      String   @unique
  color     String?
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt
}
```

Database Table Structure:
- id: TEXT PRIMARY KEY (CUID)
- name: TEXT NOT NULL UNIQUE
- slug: TEXT NOT NULL UNIQUE
- color: TEXT (nullable, for UI display)
- createdAt: DATETIME DEFAULT CURRENT_TIMESTAMP
- updatedAt: DATETIME (auto-updated)

Unique Constraints (2 total):
1. Tag_name_key - Prevents duplicate tag names
2. Tag_slug_key - Prevents duplicate slugs (URL-friendly identifiers)

SITE MODEL EXTENSIONS
=====================

New Fields Added:
1. tags: String @default("[]")
   - Stores JSON array as string
   - Default: empty array
   - Use case: Multiple tags per site

2. platforms: String @default("[]")
   - Stores JSON array as string
   - Default: empty array
   - Use case: Supported platforms (web, mobile, desktop, etc.)

3. screenshots: String @default("[]")
   - Stores JSON array as string
   - Default: empty array
   - Use case: URLs to site screenshots

4. useCases: String @default("[]")
   - Stores JSON array as string
   - Default: empty array
   - Use case: Common use cases for the tool

5. likesCount: Int @default(0)
   - Integer counter
   - Default: 0
   - Use case: Track site popularity

New Relationship:
- feedbacks: Feedback[]
  - One-to-many relationship
  - A Site can have multiple Feedback records
  - Enables feedback management per site

Updated Field:
- updatedAt: DateTime @updatedAt
  - Changed from: @default(now())
  - Now automatically updates on every record modification
  - Ensures accurate "last modified" timestamps
